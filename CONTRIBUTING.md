# Contributing to PV Tool — Community Edition

Thanks for your interest! This project is currently in its early stages, and contributions are welcome.

---

## Current Status

This project is maintained by **a single maintainer** at the moment. Contributions are appreciated, but please be patient with response times.

---

## Reporting Issues

If you find a bug or have a suggestion:

1. Check if an issue already exists
2. Open a new issue with:
   - A clear title
   - Steps to reproduce (if bug)
   - Expected vs actual behavior
   - Browser/OS info

---

## Questions / Ideas

For questions or ideas that aren't clearly bugs, open a **Discussion** instead of an issue — it's a better place for brainstorming.

---

## Contributing Code

### Before You Start

- For **small fixes** (typos, obvious bugs), feel free to open a PR directly
- For **new features or larger changes**, please open an issue first to discuss — this saves everyone time

### PR Checklist

- [ ] Tested locally (`npm run dev`)
- [ ] Build passes (`npm run build`)
- [ ] No obvious errors in browser console
- [ ] PR description explains what changed

### Process

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-feature`)
3. Commit your changes
4. Push and open a PR
5. Wait for review (maintainer is one person, so it may take a few days)

## Contributor Recognition

Contributors will be added to [CONTRIBUTORS.md](./CONTRIBUTORS.md) after their first merged PR. You can include your name in the PR description, or add yourself in a separate PR after merge.

---

## Code Style

- TypeScript: use strict typing when possible
- Components: follow existing patterns in the codebase
- Comments: add comments for non-obvious logic

If you're unsure about something, just open a draft PR and ask.

---

## Anything Else?

Open an issue or discussion — happy to help!
